# Установка и настройка Git

## Установите Git на вашем компьютере

Это можно сделать, перейдя на сайт *git-scm.com* и следуя инструкциям по установке.

## Работа с Git

Проверка наличия установленного Git
В терминале выполняем команду *git --version*, она выводит версию Git.


## Настройка Git

Ввод имени пользователя и почты:

*git config --global user.name "username"*
*git config --global user.email "useremail"*
Чтобы посмотреть изменения: *git config --list*

## Инициализация Git

Чтобы инициализировать git: *git init*

# Инструменты Git
## Git status

Выводит статус текущей ветки: *git status*


## Добавление файла к отслеживанию

Команда git add используется для добавления файлов в индекс Git. Добавьте файл в индекс с помощью команды '*git add Filename*' или добавьте все файлы с помощью '*git add .*'


## Фиксация изменений

После добавления файлов в индекс вы можете сделать коммит, который зафиксирует ваши изменения. Используйте '*git commit -m "Commit message"*', чтобы сделать коммит с сообщением.


## Информация об истории коммита

Чтобы просмотреть историю ваших коммитов, используйте команду '*git log*'. Это покажет вам автора коммита, дату и сообщение коммита.


## Сравнение с последним коммитом

Команда '*git diff*' позволяет вам увидеть различия между вашими рабочими файлами и файлами в индексе. Это позволяет вам видеть, какие изменения вы сделали, но еще не закоммитили.


## Перемещение между сохранениями

Команда git checkout используется для переключения между разными ветками и коммитами в вашем репозитории. Например, '*git checkout <branch-name>*' переключит вас на указанную ветку.


## Добавление картинок 

Чтобы добавить картинку, нужно скачать нашу картинку в нашу папку. Далее прописать в VSCode '*![наш комментарий](имя картинки)*'  


## .gitignore

Файл .gitignore используется для указания Git на файлы или директории, которые следует игнорировать и не отслеживать. Чтобы создать файл *.gitignore*, просто создайте новый текстовый файл с именем .gitignore в корневом каталоге вашего репозитория. Затем добавьте имена файлов, которые вы хотите игнорировать. Например, чтобы например игнорировать картинки png добавьте '* *.png*' чтобы все файлы .log, добавьте '* *.log*' в ваш файл *.gitignore*


## Создание ветки

Чтобы создать новую ветку, воспользуемся командой 'git branch <имя ветки>'.


## Слияние веток

Слияние веток выполняется с помощью команды git merge. Для слияния вашей текущей ветки с другой веткой используйте 'git merge <other-branch-name>'


## Решение конфликтов

При слиянии веток могут возникнуть конфликты, если одни и те же части файла были изменены в обеих ветках. Git помечает конфликтные области в файле, и вам нужно ручное решить эти конфликты. Конфликтные области будут помечены <<<<<<<, ======= и >>>>>>>. Все, что находится между <<<<<<< и ======= - это ваш код, а все, что находится между ======= и >>>>>>> - это код из другой ветки.


## Удаление ветки

Чтобы удалить ветку, используйте команду 'git branch -d <branch-name>'. Это удалит ветку, если она была слита. Если вы хотите удалить ветку, даже если она не была слита, используйте 'git branch -D <branch-name>'.


# Работа с удаленными репозиториями

## Работа с Forks

Fork репозитория — это копия репозитория, которую вы можете создать на своем аккаунте, чтобы внести изменения без воздействия на оригинальный репозиторий. Это может быть полезно, если вы хотите внести изменения в проект, но у вас нет прав на запись в оригинальный репозиторий.

## 1. Создание Fork Репозитория
На странице репозитория на веб-сайте (например, GitHub) нажмите кнопку "Fork". Это создаст копию репозитория на вашем аккаунте.

## 2. Клонирование Fork
Вы можете клонировать fork на свой локальный компьютер с помощью команды *git clone url*, где *url* - это URL-адрес вашего fork на веб-сайте.

## 3. Переход в Директорию Репозитория

Перейдите в директорию склонированного репозитория
*cd 'имя-репозитория'*

## 4. Создание Новой Ветки

Создайте новую ветку в вашем репозитории с помощью команды checkout -b.
*git checkout -b 'имя-новой-ветки'*

## 5. Создание Нового Файла

Создайте новый файл с помощью команды touch.
'bash'
*touch Git_instructions.txt* или в меню вашей папки VSCode 'Создать файл' Эта команда создаст новый текстовый файл с именем "Git_instructions.md".

## 6. Добавление Ваших Изменений

Добавьте новый файл в область подготовки с помощью команды *git add .*

## 7. Фиксация Ваших Изменений

Фиксируйте ваши изменения с помощью команды *git commit*. Эта команда фиксирует ваши изменения и использует текст в кавычках в качестве сообщения коммита.

## 8. Отправка Ваших Изменений

Наконец, вы можете отправить ваши изменения в удаленный репозиторий с помощью команды *git push origin 'имя-новой-ветки'* Замените <имя-новой-ветки> на имя ветки, которую вы создали. Эта команда отправляет ваши изменения в ваш скопированный репозиторий на удаленном сервере.

## Внесение изменений и создание Pull Request 

Вы можете внести изменения в свой fork и затем создать Pull Request в оригинальном репозитории, чтобы предложить свои изменения.

# Основные правила работы с Git

Сохраняйте свои изменения в отдельных ветках. Используйте git branch и git checkout для создания и переключения веток.

Совершайте коммиты регулярно. Используйте git commit для сохранения своих изменений.

Не меняйте историю публичных веток. Если ваша ветка была опубликована и другие люди уже работали с ней, не используйте команды, которые изменяют историю, такие как git rebase.

Производите тестирование и ревью кода перед слиянием веток. Убедитесь, что ваш код работает правильно, и он соответствует стандартам проекта.

Следите за актуальностью своей копии репозитория. Используйте git pull или git fetch и git merge, чтобы сохранить свой репозиторий актуальным по отношению к оригиналу.
